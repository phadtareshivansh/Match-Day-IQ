Platform

Built around the two things that actually go wrong on matchday

Zones drift out of status, and incidents get reported twice or not at all. MatchDay IQ keeps both in one place, polling every four seconds so nobody's screen goes stale.

◈

Zone status board

Cycle a zone through Clear → Watch → Alert with one tap. Every staff member on the network sees the change within seconds — no refresh, no group chat.

▲

Incident logging

Log type, severity, and location in seconds. The last 100 incidents stay visible to the whole crew, so nothing gets reported into a void.

◐

Personal checklists

Shift checklists and language preference are saved per account, so switching devices mid-shift doesn't mean starting over.

Roles

Your account decides what you can touch

Persona isn't a toggle you can flip on the fly — it's locked to the account that was provisioned for you, and the server enforces it on every request.

FULL CONTROL

  • ✓ Manage own checklist and language preference

Security

A real login, not a shared link

Session-backed auth

Sign-in issues an 8-hour, httpOnly session cookie — matched to a shift length, not left open indefinitely.

Per-account data isolation

Preference reads and writes check that the requested user ID matches your own session — nobody reads or overwrites someone else's saved state.

Hashed credentials

Every password is hashed with bcrypt before it touches the database. Nothing is ever stored in plain text.

POST /api/auth/login
{ "username": "organizer1", "password": "••••••••" }

→ 200 session cookie set, expires in 8h

GET /api/preferences/:userId
→ 403 if :userId ≠ session user

POST /api/incidents/:id/resolve
